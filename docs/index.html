Description

This project focus on implementing texture mapping from a 2D texture map to a 3D surface based on what we have done so far with assignment 3. There is some similarity to what we have done in assignment 1 for texture mapping, this is a 3D version of that. Original in assignment 3, the albedo was a constant, now I am going to update that value according to the texture color mapped to the surface at that location. Some ideas from assignment 1 can be adapted here. For example, normalize the coordinates in object space, pass that coordinates to the texture space to get a texel, then map that piece of texture back to the object.

Goals and Deliverables - Plan to Deliver

Planar Mapping

The planar mapping should be the easiest. Drop the z coordinate of the 3D surface. To get a texel from the texture map, convert the normalized (x, y) coordinates from the 3D space to texture space. Planar mapping is illustrated below, so the deliverable here should be some geometry with normal texture on front and back. Texture on other faces should be distorted.

Spherical Mapping

Convert the point in 3D space to spherical coordinates of a unit sphere. The texture space coordinates will also be normalized by PI and 2PI for inclination and azimuth. This kind of mapping is best for spherical objects. For any other kind of object, the texture should be distorted along the stretched or compressed radius of the sphere.

Cylindrical Mapping

Given a point in 3D space, convert to cylindrical coordinates where the radius is 1. The width of the texture map should correspond to the azimuth of the cylindrical coordinates, the height corresponds to the axial coordinate.
